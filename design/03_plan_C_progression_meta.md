# 方案C：成长向元游戏（Progression Meta System）

> 参考原型：Muse Dash（角色/道具系统）、maimai DX（段位/Rating）、
> 任何带有解锁/成长玩法的音乐游戏
>
> 定位：核心玩法仍为标准竖轨，但在外层构建成长感与目标感，
> 使玩家即使在音符技巧不足时也有明确的前进方向。

---

## 一、设计理念

> "每一次游玩都应该有意义，无论是技术提升、解锁内容，还是推进故事。"

- 音乐游戏的核心是重复练习，需要设计机制让"重复"产生价值
- 玩家应该在任何技术水平下都能感受到进步
- 社交/竞争元素（排行榜、段位）给高水平玩家长期目标

---

## 二、核心游玩层（与方案A相同）

方案C 的基础游玩层采用**方案A的标准竖轨系统**，判定/计分/血条等完全一致。

差异化来自**元游戏层（Meta Layer）**。

---

## 三、角色系统

### 3.1 角色概念
- 玩家有一名"游玩角色"，角色在游玩时出现在背景/侧边
- 角色提供**被动特性**和**主动技能**，影响游玩体验（而非游戏结果）

### 3.2 被动特性示例

| 特性名 | 效果 |
|--------|------|
| **热身** | 曲目前 15 秒判定窗口扩大 20% |
| **专注** | 当前无连击时判定窗口扩大 10%（容错更高） |
| **爆发力** | 最大连击达到 100 后，判定线出现特殊光效（纯视觉） |
| **完美主义** | 获得 PERFECT+ 时额外 +2 分 |
| **稳定器** | BAD 不断连击（只有 MISS 断连击） |

> "稳定器"这类特性会实质性影响游玩体验，需要在设计时区分
> "改变规则的特性"和"纯奖励性特性"

### 3.3 主动技能
- 每场游玩可携带**1个主动技能**，通过积累"技能槽"触发（每 50 COMBO 积累一格，满3格可用）
- 技能示例：

| 技能名 | 效果 | 冷却 |
|--------|------|------|
| **时间慢化** | 所有音符速度降低 50%，持续 5 秒 | 每局1次 |
| **生命回复** | 立刻恢复 30% 生命值 | 每局2次 |
| **双倍COMBO** | 接下来 10 秒连击数 ×2（影响结算统计） | 每局1次 |
| **全判完美** | 接下来 5 秒所有判定强制提升一档 | 每局1次 |

### 3.4 角色获取方式
- 默认解锁 1 名角色（全能型，无特性也无技能）
- 通过游玩积累的**星尘（Stardust）**货币解锁
- 特定成就解锁限定角色

## 四、成长与解锁系统

### 4.1 玩家等级（Player Level）
- 每次游玩后获得**经验值（EXP）**
- EXP 来源：游玩曲目 + 判定表现加成 + 连击奖励
- 等级上升奖励：星尘、新曲目解锁券、角色碎片

```
EXP 计算 = 基础EXP（by 曲目难度等级）
          + 清版奖励（通关 ×1.5，清版失败 ×0.5）
          + 评级奖励（S级+50EXP, A级+20EXP）
          + 首次游玩奖励 ×2
```

### 4.2 曲目解锁
曲目分为：

| 类型 | 解锁方式 |
|------|---------|
| **基础曲** | 直接可玩，永久开放 |
| **等级解锁曲** | 玩家等级达到指定值自动解锁 |
| **成就解锁曲** | 完成指定成就（如"在X歌曲获得FC"） |
| **故事解锁曲** | 推进故事章节后解锁 |
| **限时曲** | 特定日期段开放（节日活动） |

### 4.3 曲目收藏与整理
- 玩家可以为曲目添加**收藏**标记
- 自定义**播放列表**（练习列表、表演列表等）
- 历史最佳记录显示在曲目选择界面

## 五、段位系统（Rating System）

### 5.1 个人 Rating（PR）
- 每首曲目的得分对应一个"单曲 Rating"
- 玩家总 Rating = 前 N 首最高单曲 Rating 之和（N=30，类似 maimai）

```
单曲Rating = 曲目定数 × 成绩系数

成绩系数:
  SS（≥990000分） → ×1.0
  S （≥950000分） → ×0.9 + 偏移
  A （≥900000分） → ×0.8 + 偏移
  以此类推...
```

### 5.2 Rating 段位

| 段位 | Rating 要求 | 标志颜色 |
|------|------------|---------|
| 铁（Iron） | 0 ~ 499 | 灰色 |
| 铜（Bronze） | 500 ~ 999 | 铜色 |
| 银（Silver） | 1000 ~ 1999 | 银色 |
| 金（Gold） | 2000 ~ 2999 | 金色 |
| 白金（Platinum） | 3000 ~ 3999 | 白金色 |
| 钻石（Diamond） | 4000 ~ 4999 | 钻石蓝 |
| 大师（Master） | 5000+ | 紫色渐变 |

### 5.3 段位考核（挑战模式）
- 每个段位有一组"段位曲目"（3首）
- 通过条件：3首均达到指定分数线，总失误次数 ≤ 上限
- 通过后：获得段位称号、限定头像框
- 失败后可以无限次重试

## 六、成就系统

### 6.1 成就分类

| 分类 | 示例 |
|------|------|
| **游玩成就** | "游玩100首曲目"、"累计游玩时间超过10小时" |
| **技术成就** | "在任意HARD曲获得FC"、"连续3首曲目获得S级" |
| **收集成就** | "解锁全部角色"、"在所有基础曲目获得评级" |
| **隐藏成就** | 不提示条件，完成特定操作后触发（惊喜感） |

### 6.2 成就奖励
- 星尘货币
- 特殊称号（在排行榜/个人页显示）
- 限定角色/皮肤碎片

---

## 七、排行榜系统

### 7.1 排行榜层级

| 层级 | 说明 |
|------|------|
| **全球榜** | 所有玩家，按单曲最高分排列 |
| **好友榜** | 仅显示已添加好友的成绩 |
| **周榜** | 每周一重置，取本周最高分 |

### 7.2 排行榜显示信息
- 排名、玩家名、头像框、段位图标
- 最高分、准确率、达成时间
- 自己的排名在界面底部固定显示（即使不在前100内）

---

## 八、故事模式（可选章节）

### 8.1 概念
- 游戏设有主线故事，通过游玩特定曲目推进剧情
- 每章节包含：剧情对话 → 游玩曲目 → 结局对话 → 解锁奖励

### 8.2 故事与音乐结合
- 同一首曲子在不同剧情节点可有"回忆版"（降难度）和"决战版"（最高难度）
- 故事结局不同时，同一曲目的背景和视觉效果有差异

### 8.3 章节结构
- 总计 N 章（建议5~8章作为基础内容）
- 每章 3~5 首曲目
- 章节通关解锁：限定角色、额外曲目、背景皮肤

---

## 九、每日任务系统

### 9.1 每日任务
每天随机生成 3 个任务（每日重置）：

| 示例任务 | 奖励 |
|---------|------|
| 游玩任意3首曲目 | 星尘 ×30 |
| 在NORMAL难度获得A级以上 | 星尘 ×50 |
| 累计获得50次 PERFECT+ | 星尘 ×40 |
| 在任意曲目达到100连击 | EXP ×200 |

### 9.2 每周任务
每周一刷新，难度更高，奖励更丰厚：

| 示例任务 | 奖励 |
|---------|------|
| 完成任意10首曲目 | 角色碎片 ×1 |
| 在HARD难度获得任意S级 | 限定头像框 |
| 段位考核通过 | 星尘 ×200 |

---

## 十、货币与付费设计（友好型）

| 货币 | 来源 | 用途 |
|------|------|------|
| **星尘**（Stardust） | 游玩、任务、成就 | 解锁角色、皮肤、曲目 |
| **音符之石**（Note Crystal） | 极少量游玩奖励 + 可选购买 | 限定内容加速解锁 |

> 设计原则：所有游戏内容均可通过游玩获得，付费仅加速进度，
> 不存在"花钱才能变强"的机制。

---

## 十一、方案C 优缺点评估

### 优点
- 给所有技术水平的玩家提供长期目标
- 故事模式提升代入感，降低纯技术压力
- 角色/技能系统增加策略性和个性化
- 每日任务确保玩家持续登录动力

### 缺点
- 系统复杂，初期开发量远大于方案A/B
- 元游戏设计不当可能喧宾夺主，分散核心玩法注意力
- 排行榜和段位系统需要服务端支持（单机版无法实现）
- 角色技能平衡性需要大量测试

### 适用场景
- 目标为长期运营的手机/PC 游戏
- 社区驱动、希望玩家长期留存
- 有服务端开发能力的团队
