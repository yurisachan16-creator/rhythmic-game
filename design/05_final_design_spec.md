# 最终综合设计方案

> 基于决策：PC端 / 支持UGC / 弱联网 / 4K起步预留扩展 / 由浅入深

---

## 一、定位总结

| 维度 | 决策 | 设计影响 |
|------|------|---------|
| 平台 | PC 键盘为主 | SLIDE 用"跨键滑动"实现，FLICK 降优先级 |
| 谱面 | 支持 UGC | 谱面格式必须开放标准化，需要配套编辑器设计 |
| 联网 | 弱联网/单机 | 所有核心功能本地化，排行榜/同步为可选附加 |
| 键位 | 4K 起步 | 输入系统抽象为"N轨通用"，先实现4K |
| 基调 | 由浅入深 | 完善难度分层，宽松 EASY 模式，新手引导必要 |

---

## 二、核心游玩系统

### 2.1 音符类型（PC键盘优化版）

| 优先级 | 类型 | 操作 | 说明 |
|--------|------|------|------|
| P0（必做） | **TAP** | 按键瞬间 | 最基础单键 |
| P0（必做） | **HOLD** | 按住到尾部 | 长按长条 |
| P1（近期） | **CHORD** | 同帧多键同时按 | 和弦，仅是排布标记，不改变判定逻辑 |
| P2（中期） | **SLIDE** | 按住A键再按B键，松A | 键盘版跨轨滑动，两个轨道间的连接线 |
| P3（后期） | **RELEASE** | 在指定时机松键 | 高难度专用，区分于 HOLD 尾部判定 |

> FLICK（快拨）：键盘上体感差，**不做**，以 TAP 代替。

---

### 2.2 判定系统

**判定档位：**

| 档位 | 偏差窗口 | 默认颜色 | COMBO影响 |
|------|---------|---------|----------|
| PERFECT+ | ±16ms | 金色 | 维持 |
| PERFECT | ±40ms | 蓝白 | 维持 |
| GREAT | ±75ms | 绿色 | 维持 |
| GOOD | ±110ms | 黄色 | 维持 |
| BAD | ±150ms | 红色 | **断连击** |
| MISS | 超出 / 未按 | 灰色 | **断连击** |

> "由浅入深"的体现：EASY模式下所有窗口乘以 **×1.5**，降低精度要求

**判定偏移校正（Offset）：**
- 玩家可设置全局判定偏移 -200ms ~ +200ms，校正音响/显示延迟
- 支持"自动校正引导"：游玩校正谱时系统统计平均偏差并建议值

---

### 2.3 计分系统

**基础分制：满分 1,000,000**

```
每音符基础分 = 1,000,000 ÷ 谱面总音符当量
（HOLD 计为 2 当量：头部 + 尾部各 1）
（SLIDE 计为 2 当量：起点 + 终点各 1）
```

**判定权重：**

| 判定 | 得分倍率 |
|------|---------|
| PERFECT+ | ×1.005（超出部分可使总分突破1,000,000） |
| PERFECT | ×1.0 |
| GREAT | ×0.7 |
| GOOD | ×0.4 |
| BAD | ×0.2 |
| MISS | ×0 |

**准确率（独立统计）：**
```
ACC% = Σ(各判定分值权重 × 该判定数量) / (总音符当量 × 1.0) × 100
```

**连击：** 不影响实时分数，仅影响结算评级和成就判断

---

### 2.4 血条模式

进入曲目前选择（**默认 NORMAL**）：

| 模式 | 初始血量 | MISS扣血 | BAD扣血 | PERFECT回血 | 清版 |
|------|---------|---------|---------|------------|------|
| **EASY** | 70% | 3% | 0.5% | 0.5% | 是 |
| **NORMAL** | 50% | 5% | 1% | 0.5% | 是 |
| **HARD** | 100% | 10% | 2% | 0% | 是（立刻） |
| **NO FAIL** | — | — | — | — | 否 |

> 由浅入深：新手默认进入 EASY 模式，可在设置中永久改变默认值

---

### 2.5 结算系统

**结算数据展示：**
- 最终分数 + 是否 PB（个人最佳）
- 准确率（ACC%）
- 最大连击
- 各判定计数（P+/P/GR/GD/BD/MS）
- 特殊标记：AP / FC / GFC

**字母评级（基于分数）：**

| 评级 | 分数线 |
|------|--------|
| S+ | ≥1,000,000（全P+满分） |
| S | ≥950,000 |
| A | ≥900,000 |
| B | ≥800,000 |
| C | ≥700,000 |
| D | ≥500,000 |
| F | <500,000 或清版 |

## 三、键位系统（可扩展设计）

### 3.1 当前阶段：4K

| 轨道 | 默认键位 | 备用键位 |
|------|---------|---------|
| 1 | D | ← |
| 2 | F | ↓ |
| 3 | J | ↑ |
| 4 | K | → |

### 3.2 扩展预留（后续）

| 模式 | 轨道 | 追加键位 |
|------|------|---------|
| 5K | +1 | 空格居中 |
| 6K | +2 | S / L |
| 7K | +3 | A / ; / 空格 |
| 8K | +4 | A / S / L / ; |

> 设计要点：**键位系统必须从一开始就抽象为"N条轨道→N个键"的配置驱动模式**，
> 不写死4K逻辑，以便后续只改配置文件即可支持新键位模式。

### 3.3 键位配置（玩家可自定义）
- 每个轨道可绑定最多 2 个键（主键 + 副键）
- 配置存储在本地存档中

## 四、谱面格式规范（UGC标准）

### 4.1 文件结构
谱面以**单一目录**为单位打包，目录结构如下：

```
[歌曲名]/
  ├── song.ogg          # 音频文件（推荐 OGG）
  ├── cover.png         # 封面图（推荐 512×512）
  ├── meta.json         # 元数据
  └── charts/
       ├── easy.json    # 简单难度谱面
       ├── normal.json  # 普通难度谱面
       ├── hard.json    # 困难难度谱面
       └── expert.json  # 专家难度谱面（可选）
```

### 4.2 meta.json 字段定义

```json
{
  "title": "歌曲标题",
  "title_unicode": "标题（原语言，若有）",
  "artist": "艺术家",
  "artist_unicode": "艺术家（原语言，若有）",
  "bpm": 140,
  "audio_file": "song.ogg",
  "cover_file": "cover.png",
  "preview_start": 45.0,
  "charts": [
    { "difficulty": "EASY",   "file": "charts/easy.json",   "level": 3 },
    { "difficulty": "NORMAL", "file": "charts/normal.json", "level": 6 },
    { "difficulty": "HARD",   "file": "charts/hard.json",   "level": 10 }
  ]
}
```

### 4.3 谱面文件（chart.json）字段定义

```json
{
  "version": 1,
  "key_count": 4,
  "od": 8,
  "bpm_events": [
    { "beat": 0, "bpm": 140 }
  ],
  "scroll_events": [
    { "beat": 0, "speed": 1.0 }
  ],
  "notes": [
    {
      "type": "TAP",
      "lane": 0,
      "beat": 1.0
    },
    {
      "type": "HOLD",
      "lane": 2,
      "beat": 3.0,
      "end_beat": 4.5
    },
    {
      "type": "SLIDE",
      "lane_start": 0,
      "lane_end": 3,
      "beat": 5.0,
      "end_beat": 5.5
    }
  ],
  "events": [
    { "beat": 8.0, "type": "LANE_HIGHLIGHT", "lane": -1, "color": "#FFD700", "duration_beats": 4 }
  ]
}
```

### 4.4 字段说明

| 字段 | 说明 |
|------|------|
| `od` | Overall Difficulty，0~10，控制判定窗口严格度 |
| `beat` | 以"拍"为单位的时间位置（浮点数，0 = 曲目起始拍） |
| `bpm_events` | 变BPM列表，每项指定从某拍开始的新BPM |
| `scroll_events` | 变速列表，控制视觉滚动速度（不影响判定时机） |
| `events` | 谱面演出事件（高亮、震动等），可为空 |
| `lane` | 轨道编号，从0开始（0~key_count-1） |
| `lane: -1` | 全轨道 |

### 4.5 拍位到时间的转换规则
```
时间(ms) = 初始偏移 + Σ(每段BPM区间的拍数 × 60000 / BPM)
```
变BPM处理：遍历 bpm_events，逐段计算到目标拍位的毫秒累计值。

## 五、难度系统（由浅入深）

### 5.1 难度等级数字化
- 每张谱面有一个 **Level 1~15** 的数字等级
- 等级由谱面制作者标注，社区可以通过评分反馈调整
- 选歌界面可按等级筛选、排序

### 5.2 难度标签体系

| 标签 | Level范围 | 说明 |
|------|----------|------|
| EASY | 1~4 | 单指练习，无密集段落 |
| NORMAL | 5~7 | 双手协调，节奏规律 |
| HARD | 8~10 | 变速、密集、需要技巧 |
| EXPERT | 11~13 | 高手向，强手速要求 |
| SPECIAL | 14~15 | 超高难或演出特化谱 |

### 5.3 新手引导流程（Game Design层面）

```
第一次启动
  → 键位设置引导（展示4个轨道，逐个按键确认）
  → 偏移校正引导（播放节拍音，玩家按拍，自动建议偏移值）
  → 教程谱（Level 1，极低速度，仅 TAP，有按键提示高亮）
  → 第一首正式曲（Level 2~3，默认EASY血条，默认NO FAIL可选）
```

### 5.4 难度推荐系统
- 曲目详情页显示"推荐Level范围"（基于玩家当前平均成绩自动判断）
- 对超出玩家当前水平的谱面显示"挑战"标记，对已掌握的显示"已通关"状态

## 六、本地存档设计

### 6.1 存档内容

| 数据 | 说明 |
|------|------|
| 玩家设置 | 键位、偏移、滚速、音量、血条模式默认值 |
| 游玩记录 | 每首谱面的最高分、最高ACC%、最高COMBO、评级、FC/AP标记 |
| 成就进度 | 已解锁成就列表、进行中成就进度 |
| 解锁状态 | 已解锁曲目列表、角色（如有） |
| 统计数据 | 总游玩次数、总游玩时长、各判定总计数 |

### 6.2 弱联网同步（可选）
- 玩家可选择绑定账号（邮箱+密码）
- 绑定后：本地存档自动备份到云端
- 联网时：拉取全球/好友排行榜数据（只读）
- 离线时：仅显示本地记录，排行榜显示"离线"提示
- **不强制联网**，所有游玩功能离线可用

## 七、成就系统（本地化，P1优先级）

### 7.1 初期成就清单（MVP阶段）

| 成就 | 条件 | 奖励 |
|------|------|------|
| 初次体验 | 完成第一首曲目 | 称号"初音者" |
| 完美偏执 | 任意曲目获得 AP | 称号"完美主义者" |
| 百连不断 | 任意曲目达到 100 COMBO | 解锁新皮肤 |
| 全连奏者 | 任意曲目获得 FC | 称号"不断奏者" |
| 进阶之路 | 在 HARD 难度获得 S 级 | 解锁额外曲目 |
| 千曲之旅 | 总游玩次数达到100 | 称号"老玩家" |
| 偏移大师 | 完成偏移校正引导 | 特效皮肤 |

### 7.2 成就对游玩的引导作用
- 成就设计遵循"引导玩家尝试新功能"原则
- 每个新系统（SLIDE音符、变速段落等）上线时配套成就

## 九、谱面编辑器功能设计（Phase 3）

> 编辑器是 UGC 生态的核心，以下为最小可用版本的功能清单

### 9.1 基础功能
- 加载音频文件，显示波形
- 设置 BPM，显示节拍格线
- 在指定拍位/轨道放置 TAP 音符
- 拖拽设置 HOLD 音符长度
- 播放预览（边听边校对）
- 保存为标准 chart.json

### 9.2 进阶功能
- 多 BPM 段支持
- 变速（scroll_events）编辑
- SLIDE 音符路径编辑
- 事件轨道（高亮/震动等）

### 9.3 编辑器不需要做的（范围控制）
- 不内置音频编辑
- 不内置封面设计
- 不内置谱面审核/上传（弱联网阶段以手动分享目录为主）
