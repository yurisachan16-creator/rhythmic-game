# 方案A：标准竖轨式（Classic Vertical Lane）

> 参考原型：DJMAX Respect、osu!mania、Beatmania IIDX
>
> 定位：玩法经典易上手，核心机制成熟，适合作为基础版本

---

## 一、核心游戏循环

```
谱面加载
  → 音符从上方按拍位生成，向下滚动
    → 音符到达判定线时，玩家按对应键位
      → 系统计算时间偏差，给出判定结果
        → 更新分数、连击、血条
          → 曲目结束 → 结算界面
```

---

## 二、轨道系统

### 键位模式
支持多种键位模式，同一套核心逻辑适配：

| 模式 | 轨道数 | 推荐输入 |
|------|--------|---------|
| 4K | 4条 | 键盘 D F J K |
| 6K | 6条 | 键盘 S D F J K L |
| 7K | 7条 | 键盘 S D F 空格 J K L |
| 8K | 8条 | 键盘 A S D F J K L ; |

### 轨道布局
- 轨道等宽垂直分布，占屏幕中央约 60% 宽度
- 两侧留空用于显示 UI（分数、连击、血条）
- 判定线位于屏幕下方约 15% 处（可在设置中微调）
- 音符从屏幕顶部（或顶部以外）出现，向判定线滚动

## 三、音符类型

### 3.1 基础音符
| 类型 | 名称 | 操作 | 判定逻辑 |
|------|------|------|---------|
| TAP | 单击音符 | 按键瞬间触发 | 音符中心到达判定线时，计算按键时间偏差 |
| HOLD | 长按音符 | 按住并在尾部松开 | 头部：同TAP判定；尾部：松键偏差单独计算 |
| RELEASE | 抬键音符 | 松开按键 | 需要在指定时机松键（高难度专用） |

### 3.2 HOLD 音符细则
- 头部判定：与TAP相同，偏差越小评级越高
- 持续段：按住即满分，提前松键触发 BREAK（降低评级）
- 尾部判定：松键时机 ±判定窗口内计算
- 若头部 MISS，HOLD 整体视为 MISS，不可补救

---

## 四、判定系统

### 4.1 判定档位

| 档位 | 英文 | 偏差范围 | 说明 |
|------|------|---------|------|
| MAX PERFECT | MAX | ±16ms | 最高精度，通常显示不同颜色特效 |
| PERFECT | PF | ±40ms | 主要高精度区间 |
| GREAT | GR | ±75ms | 普通好 |
| GOOD | GD | ±110ms | 勉强过关 |
| BAD | BD | ±150ms | 很差，通常断连击 |
| MISS | MS | 超出BAD或未按 | 断连击，扣大量血量 |

> 所有窗口值均可在谱面配置中按曲目难度单独设定

### 4.2 判定优先级规则
- 单帧内多个音符同时在窗口内：优先判定最近的一个
- 同轨道提前按键（过早）：若在 BAD 窗口前沿则算 BAD，否则无效
- 同轨道重叠音符：前一个未离开窗口时，新音符暂不激活

### 4.3 输入检测
- 仅响应**按下**（Key Down）事件
- 长按音符同时检测**持续按住**和**松开**（Key Up）事件
- 同一帧内允许多个轨道同时触发（和弦支持）

---

## 五、计分系统

### 5.1 基础分制
- 满分：**1,000,000 分**
- 每个音符的基础分 = 1,000,000 ÷ 谱面总音符数
  - HOLD 音符按"2个音符"计算（头 + 尾各占1份）

### 5.2 判定权重

| 判定 | 得分倍率 |
|------|---------|
| MAX PERFECT | ×1.005（超出满分部分累积为奖励分） |
| PERFECT | ×1.0 |
| GREAT | ×0.7 |
| GOOD | ×0.4 |
| BAD | ×0.2 |
| MISS | ×0 |

> MAX PERFECT 的超出分数最终累积，可使满分玩家获得最高 **1,005,000 分**（类似DJMAX的MAX评价体系）

### 5.3 准确率（Accuracy）
```
ACC = (MAX×1 + PF×1 + GR×0.7 + GD×0.4 + BD×0.2) / (总音符数 × 1) × 100%
```
准确率与分数独立显示，共同决定最终评级。

### 5.4 连击（COMBO）
- GOOD 及以上维持连击
- BAD 或 MISS 断连击归零
- 连击仅影响结算显示，**不影响实时分数**（避免雪球效应）

## 六、血条系统

### 6.1 模式选择
玩家可在进入曲目前选择血条模式：

| 模式 | 说明 |
|------|------|
| **NORMAL** | 初始 50% 血量，MISS 扣 5%，GOOD/BAD 扣 1%，PERFECT 回 0.5% |
| **EASY** | 初始 50% 血量，扣血减半，回血加倍；提供"EASY"标记 |
| **HARD** | 初始 100% 血量，MISS 扣 10%，任意时刻降至 0% 即立刻清版 |
| **NO FAIL** | 无血条，不可清版，结算时显示实际表现 |

### 6.2 清版（Stage Failed）
- NORMAL/HARD 模式下血量降至 0% 触发
- 立刻停止音乐，进入失败结算界面
- 显示已达到的分数和判定统计

---

## 七、结算系统

### 7.1 结算数据
- 最终分数（含MAX奖励）
- 准确率（%）
- 连击情况：最大连击、是否全连（FC）、是否全MAX（AP/MAX）
- 判定详情：各档位计数
- 是否新纪录（Personal Best）

### 7.2 字母评级

| 评级 | 条件 |
|------|------|
| **S+** | 分数 ≥ 1,000,000（即 MAX PERFECT 满分） |
| **S** | 分数 ≥ 950,000 |
| **A** | 分数 ≥ 900,000 |
| **B** | 分数 ≥ 800,000 |
| **C** | 分数 ≥ 700,000 |
| **D** | 分数 ≥ 500,000 |
| **F** | 分数 < 500,000 或清版 |

### 7.3 特殊标记

| 标记 | 条件 |
|------|------|
| **AP**（All Perfect） | 所有音符为 MAX 或 PERFECT |
| **FC**（Full Combo） | 无 MISS，但允许 BAD |
| **GFC**（Good Full Combo） | 无 MISS 且无 BAD，允许 GOOD 及以上 |

## 八、谱面系统

### 8.1 谱面元数据
- 曲目标题、艺术家、封面
- BPM（可含变速标记）
- 难度名称（EASY / NORMAL / HARD / EXPERT 等）
- 难度等级（数字，如 1~15）
- 谱面作者

### 8.2 时间轴机制
- 所有音符以**小节号 + 拍号偏移**记录（如：小节3，第2拍，偏移1/4拍）
- 支持变 BPM（在特定小节切换 BPM）
- 支持变速（SCROLL SPEED，影响视觉速度，不影响实际判定时机）
- 玩家可设置个人滚速倍率（× 0.5 ~ × 5.0）

### 8.3 谱面难度设计分档

| 难度 | 说明 | 示例音符密度 |
|------|------|------------|
| EASY | 单指操作，无密集复杂型 | 1~2 note/s |
| NORMAL | 双手基础协调 | 3~5 note/s |
| HARD | 需要较强手速和协调 | 5~8 note/s |
| EXPERT | 高密度、长条密集、手速流 | 8~14 note/s |

## 九、游戏设置（玩家可调项）

| 设置项 | 范围 | 说明 |
|--------|------|------|
| 滚速（Scroll Speed） | 0.5x ~ 5.0x | 音符下落速度 |
| 判定线偏移（Offset） | -200ms ~ +200ms | 校正个人延迟 |
| 下落方向 | 上→下 / 下→上 | 可翻转方向 |
| 按键映射 | 自定义 | 各轨道绑定键 |
| 音效音量 | 0~100% | 打击音效 |
| 音乐音量 | 0~100% | BGM音量 |
| 皮肤选择 | 多套 | 音符外观、判定线外观 |

## 十、方案A 优缺点评估

### 优点
- 玩法成熟，学习曲线平滑
- 谱面制作标准明确，社区易于贡献
- 可无缝扩展键位模式（4K/6K/7K/8K）
- 判定系统可精细调整，适配不同水平玩家

### 缺点
- 视觉表现较为传统，难以脱颖而出
- 音符类型较少（仅TAP+HOLD），表现力有限
- 对非核心向玩家吸引力偏低

### 适用场景
- 作为**基础框架**，可以在此之上叠加方案B/C的特色系统
- 适合先实现核心后迭代扩展
