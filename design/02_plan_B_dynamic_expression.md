# 方案B：动态表现式（Dynamic Expression System）

> 参考原型：Sound Voltex、Arcaea、Chunithm（取各自特色机制）
>
> 定位：在标准竖轨基础上，加入丰富的音符类型和动态谱面特效，
> 让谱面本身具备"演出感"，玩家在游玩时同时体验视觉叙事。

---

## 一、设计理念

> "谱面不只是节拍的映射，而是歌曲情感的可视化表达。"

- 谱面设计者可以控制轨道宽窄、速度分段、视觉特效触发
- 不同段落（副歌/间奏/高潮）有截然不同的视觉风格
- 音符种类足够丰富，使简单曲目也有表现空间

---

## 二、轨道系统（扩展版）

### 2.1 基础轨道
- 默认 4 条主轨（Main Lanes）
- 主轨宽度统一，排列于屏幕中央

### 2.2 动态轨道特性（谱面可控）
谱面可以在时间轴上指定以下轨道变化：

| 特性 | 说明 | 示例用法 |
|------|------|---------|
| **轨道高亮** | 特定轨道在某段时间变色 | 副歌段高亮全部轨道 |
| **轨道隐藏** | 特定轨道在某段时间透明（音符仍需打） | 制造挑战性 |
| **速度分段** | 特定时段音符加速或减速 | 慢速段→急加速高潮 |
| **判定线移动** | 判定线在纵向位置缓慢移动 | 视觉演出，不影响判定逻辑 |
| **屏幕震动** | 配合音效的震动效果 | BOSS段落 |

> 以上均为**纯视觉特效**，不改变判定时机，玩家可在设置中关闭视觉特效

---

## 三、音符类型（扩展版）

### 3.1 所有音符类型

| 类型 | 名称 | 操作 | 特色 |
|------|------|------|------|
| TAP | 单击 | 按键瞬间 | 基础音符 |
| HOLD | 长按 | 按住到尾部 | 基础长条 |
| SLIDE | 横向滑动 | 从一个轨道滑至另一轨道 | 跨轨道连续滑动 |
| FLICK | 快拨 | 按下后立刻松开（区别于长按） | 需要"弹拨"感 |
| CHORD | 和弦 | 同时按多键 | 谱面显示时并排高亮 |
| ECHO | 回响音符 | 顺序快速按同一键 | 同轨连续2~4个极短间隔音符 |

### 3.2 音符详细设计

#### SLIDE（横向滑动）
- 从起始轨道延伸出一条曲线/直线，终点在另一轨道
- 操作：在起始轨道按下，**持续按住**并移动手指/鼠标至终点轨道，在终点松开
- 键盘操作：按住起始键，再按下终点键，松开起始键
- 判定：起点按下时机 + 终点到达时机
- 视觉：轨道间有高亮连接弧线

#### FLICK（快拨）
- 外观：带有上/下箭头的特殊TAP音符
- 操作：按下立刻松开（按住超过 80ms 视为错误操作，降一档判定）
- 设计目的：区分"弹奏感"与"持续感"，对应音效中的拨弦、打击感

#### ECHO（回响）
- 外观：同一轨道内 2~4 个极短间距的小音符（间隔 50~100ms）
- 玩家需要快速连点同一键
- 全部打中给额外 BONUS 分
- 设计目的：对应音效中的颤音、滚奏

#### CHORD（和弦）
- 并非独立音符类型，而是多个音符的**组合标记**
- 谱面中被标记为 CHORD 的一组音符要求同一时间内全部触发
- 全部触发：正常各自判定
- 部分触发：未触发的视为 MISS，已触发的正常给分

## 四、判定系统

### 4.1 判定档位（同方案A，微调）

| 档位 | 偏差范围 | 显示颜色 | 效果 |
|------|---------|---------|------|
| **PERFECT+** | ±16ms | 金色 | 特殊光效粒子 |
| **PERFECT** | ±40ms | 蓝白色 | 标准特效 |
| **GREAT** | ±75ms | 绿色 | 简单特效 |
| **GOOD** | ±110ms | 黄色 | 无特效 |
| **BAD** | ±150ms | 红色 | 警告效果 |
| **MISS** | 超出 | 灰色 | 音符消失，连击归零 |

### 4.2 音符专属判定规则

| 音符类型 | 额外规则 |
|---------|---------|
| HOLD | 提前松键：降一档；过晚松键：不惩罚 |
| SLIDE | 中途断开超过 100ms：尾部视为 MISS |
| FLICK | 按住超过 80ms：降一档判定 |
| ECHO | 部分打中：打中的正常判定，剩余视为 MISS |

## 五、计分系统

### 5.1 基础分制
- 满分：**1,000,000 分**（基础）
- PERFECT+ 奖励分：最多额外 **50,000 分**（满 PERFECT+ 时）
- ECHO BONUS：ECHO 全中额外每组 +500 分

### 5.2 表达分（Expression Score）
本方案新增"**表达分**"系统：

- 表达分与主分数分离，范围 0~100
- 计算方式：（PERFECT+ 数 + PERFECT 数）/ 总音符数 × 100
- 表达分影响：
  - 结算时的视觉演出质量（烟花/粒子效果密度）
  - 成就解锁门槛之一
  - **不影响字母评级**

### 5.3 连击加成
- 每达到 100 COMBO 里程碑，当前轨道出现视觉特效
- 50 COMBO 前：分数倍率 ×1.0
- 50~200 COMBO：分数倍率 ×1.01
- 200 COMBO+：分数倍率 ×1.02
- 断 COMBO 后重新计算（COMBO 倍率缓慢衰减，不立刻归零）

## 六、血条系统

### 6.1 双轨血条
本方案使用**双重血条**设计：

| 血条 | 说明 |
|------|------|
| **生命条**（Life） | 传统扣血机制，降至0清版 |
| **燃烧条**（Burst Gauge） | 高精度击打时积累，满后触发 BURST 状态 |

### 6.2 BURST 状态
- 燃烧条积满后自动触发，持续 10 秒
- 期间所有音符的判定窗口扩大 ×1.3
- 分数倍率 ×1.05
- 视觉：整个游玩区域出现特殊背景特效
- 设计目的：给玩家"爽点"，减少高难度段落的挫败感

## 七、谱面系统（扩展版）

### 7.1 谱面事件系统
谱面文件除音符外，还可以包含**谱面事件（Chart Events）**：

| 事件类型 | 参数 | 说明 |
|---------|------|------|
| BPM_CHANGE | 新BPM值 | 变速 |
| SCROLL_CHANGE | 倍率 | 视觉滚速（不影响判定） |
| LANE_HIGHLIGHT | 轨道ID、颜色、持续时间 | 轨道高亮 |
| LANE_HIDE | 轨道ID、持续时间 | 轨道隐藏 |
| CAMERA_SHAKE | 强度、时间 | 屏幕震动 |
| BURST_FORCE | 无 | 立即触发BURST（剧情谱专用） |
| BACKGROUND_EVENT | 事件ID | 触发背景动画 |

### 7.2 谱面难度分层

| 难度 | 特色 |
|------|------|
| EASY | 仅 TAP + HOLD，速度稳定 |
| NORMAL | 加入 SLIDE，少量 CHORD |
| HARD | 加入 FLICK、ECHO，变速段落 |
| EXPERT | 全类型混用，隐藏轨道，极端变速 |
| SPECIAL | 谱面本身作为演出，事件密集 |

## 八、方案B 优缺点评估

### 优点
- 谱面表现力极强，适合将"音符编排"本身作为艺术
- BURST 系统给玩家正向反馈，减少挫败感
- 双血条系统兼顾新手保护和高手挑战
- 谱面事件系统使每首歌都有独特体验

### 缺点
- 谱面制作成本高，事件系统需要专用编辑器支持
- ECHO/FLICK 等特殊音符在键盘上操作感受较差（更适合触屏）
- 视觉特效过多可能干扰判定专注度

### 适用场景
- 强调单人沉浸体验的剧情/演出向音乐游戏
- 每首歌精心制作谱面的精品路线
